# obookmark 安全与多语言升级设计规格书

## 1. 目标
对 `obookmark` 扩展进行功能增强，实现界面语言自动适配（中/英），并强化 WebDAV 凭据在 UI 界面和本地存储中的安全性。

## 2. 详细设计

### 2.1 国际化 (i18n) 方案
- **架构：** 使用 Chrome 扩展标准 i18n 机制。
- **资源：** 
    - `_locales/en/messages.json`: 英文资源。
    - `_locales/zh_CN/messages.json`: 中文(简体)资源。
- **实现：**
    - 在 `manifest.json` 中设置 `"default_locale": "zh_CN"`。
    - HTML 元素使用 `data-i18n` 属性标记。
    - `popup.js` 启动时遍历 `data-i18n` 元素，调用 `chrome.i18n.getMessage()` 进行替换。

### 2.2 隐私安全强化方案 (安全锁屏)
- **UI 防窥视：**
    - **锁定状态：** 保存后，用户名显示为前 3 位 + 星号（如 `adm***`），密码输入框保持隐藏。输入框设为 `readOnly`，并禁用文本选中（`user-select: none`），防止直接复制。
    - **解锁交互：** 增加“🔓”图标按钮。点击后，输入框恢复可编辑状态并显示完整内容，图标切换为“🔒”。
- **存储安全：**
    - **数据混淆：** 在存入 `chrome.storage.local` 之前，使用 Base64 结合一个简单的静态“盐值”对用户名和密码进行混淆处理，确保存储在本地的数据不是明文。
    - **解密读取：** 仅在执行同步（`fetch`）前瞬间进行解密还原。

## 3. 涉及的文件变更
- **`manifest.json`**: 添加 `default_locale`。
- **`_locales/`**: 新增中英语言包。
- **`popup/popup.html`**: 增加 i18n 属性，修改布局以支持“解锁”按钮。
- **`popup/popup.css`**: 添加锁定状态样式、禁用选中样式及图标样式。
- **`popup/popup.js`**:
    - 实现 i18n 自动加载逻辑。
    - 实现用户名掩码逻辑。
    - 实现锁定/解锁状态切换逻辑。
    - 实现数据存储前的混淆与读取后的还原逻辑。
- **`background.js`**: 更新自动同步逻辑，支持解密混淆后的数据。

## 4. 约束与安全警告
- 虽然本地混淆能防止肉眼直视和初级的数据提取，但开发者模式下的高级用户仍能通过代码逆向找到逻辑。本项目定位为“个人工具级安全”，不作为企业级密码管理器使用。
